# Metasploitable2 Penetration Testing Project

## 📌 Overview
This project is a hands-on penetration testing assessment performed on the intentionally vulnerable machine **Metasploitable2**.  
The goal was to simulate real-world attack scenarios, identify security weaknesses, and document vulnerabilities following a structured methodology.

---

## 🎯 Objectives
- Perform reconnaissance and information gathering
- Identify open ports and exposed services
- Detect and analyze security vulnerabilities
- Understand exploitation techniques in a controlled environment
- Document findings with remediation recommendations

---

## 🛠 Methodology

The following penetration testing phases were applied:

### 1. Reconnaissance
- Passive and active information gathering
- Target identification and initial analysis

### 2. Scanning & Enumeration
- Port scanning using Nmap
- Service and version detection
- Enumeration of exposed services

### 3. Vulnerability Analysis
- Identification of misconfigurations
- Detection of outdated and vulnerable services
- Analysis of weak authentication mechanisms

### 4. Exploitation (Theoretical / Practical depending on scope)
- Testing known vulnerabilities
- Understanding exploit behavior using security tools

### 5. Reporting
- Documenting findings in a structured security report
- Providing mitigation and security recommendations

---

## 🔧 Tools Used
- Nmap  
- Metasploit Framework  
- Manual Testing Techniques  
- Linux Environment Tools  

---

## 📊 Key Findings
- Exposed and unnecessary open ports
- Vulnerable and outdated services
- Weak system configurations
- Potential attack vectors identified through enumeration
